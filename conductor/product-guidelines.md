# 产品准则 (Product Guidelines)

## 1. 文案风格 (Prose Style)
- **简洁高效：** 应用内的提示、标签和说明应直接、明确。避免使用冗长的句子或非必要的修饰语。
- **操作导向：** 文案应侧重于引导用户完成特定任务，如“确认加班”、“导出报表”。

## 2. 视觉与 UI 风格 (Visuals & UI)
- **简约工业风：** 整体视觉应保持简洁、清爽，侧重于数据内容的清晰展示。
- **配色与间距：** 延续现有系统的色彩方案，使用充足的间距和清晰性边框，确保信息层级分明。

## 3. 用户体验原则 (UX Principles)
- **极致效率 (Efficiency First)：** 优化常用操作流程，确保用户能在最少的点击次数内完成加班状态的录入和部门切换。
- **防错与反馈 (Error Prevention)：** 在执行关键操作（如删除员工、清空数据）前，应提供清晰的二次确认和即时的结果反馈。
- **响应式一致性 (Responsive Consistency)：** 确保系统在桌面端和移动端浏览器上均能提供流畅、一致的交互体验，特别是表格展示和状态修改。

## 4. 开发与协作规范 (Development Norms)
- **中英结合 (Mixed CN/EN)：** 代码命名使用标准英文。业务逻辑、复杂算法或项目文档中应使用中文注释，以便团队内部快速理解和维护。
- **遵循官方规范 (Official Guides)：** 后端开发严格遵循 PEP 8 和 FastAPI 官方最佳实践；前端开发严格遵循 Vue 3 风格指南（Vue 3 Style Guide）。
