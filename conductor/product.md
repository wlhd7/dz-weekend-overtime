# Initial Concept

基于 FastAPI + Vue 3 的周末加班管理系统，按部门隔离管理员工名单，支持周六/周日加班状态追踪及跨部门统计。

# 产品指南 (Product Guide)

## 1. 产品愿景 (Product Vision)
本项目是一个基于 FastAPI 和 Vue 3 的轻量级、高效的周末加班人员管理与统计系统。旨在通过部门隔离、状态追踪（周六/周日/出差）及跨部门汇总统计，简化加班管理流程，确保数据的准确性和时效性。

## 2. 目标用户 (Target Audience)
- **部门负责人/班组长：** 负责各部门加班人员的日常录入、状态更新及部门内部的数据确认。
- **人事/行政管理人员：** 进行跨部门的加班数据汇总、核对及报表生成工作。

## 3. 核心功能 (Core Features)
- **核心功能 (Core Features)**
- **部门隔离管理：** 基于 Cookie (`department`) 实现部门信息的隔离管理，确保不同部门管理各自的员工名单。
- **多状态追踪：** 分别管理员工在周六、周日的加班状态（无加班、正常加班、出差）。
- **操作记录与报表控制：** 
    - 记录各部门的每日操作行为。
    - 在生成报表时，若某部门当天未进行任何操作，则报表中该部门将不予显示（无论该部门是否已有加班记录）。
- **跨部门汇总统计：** 提供全局统计页，按部门汇总周六/周日的详细名单。

## 4. 关键约束与原则 (Constraints & Principles)
- **架构兼容性：** 必须保持与现有 FastAPI 后端、Vue 3 前端及 SQLite 数据库的完全兼容。
- **UI 一致性：** 保持现有 UI 风格，确保用户操作逻辑的一致性。
- **高精度时间管理：** 确保加班数据的统计必须精确到具体的时间段和日期。
